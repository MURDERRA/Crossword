# Генератор мини-кроссворда

Генерирует заполненные сетки мини-кроссворда в стиле NYT Mini по заданному макету. Все горизонтальные и вертикальные слова берутся из словаря, ни одно слово не повторяется дважды, все пересечения согласованы.

## Требования

- Node.js (внешние зависимости отсутствуют)
- `words.txt` - по одному слову на строку, в той же папке, что и `crossword.js`. Был взят напрямую [отсюда](https://drive.google.com/file/d/1A-L4ye_VZfxSSylKO7JTyCl8nrcL0F0r/view)

## Использование

Макет сетки передаётся через stdin или как аргумент-файл:

```sh
# через stdin
echo "_____
_____
_____
_____
_____" | node crossword.js

# через файл
node crossword.js grid.txt
```

## Формат сетки

- `_` или любой символ кроме `*` - белая (заполняемая) клетка
- `*` - чёрная (заблокированная) клетка
- Размер сетки: от 1×1 до 7×7
- Слова образуются только из последовательностей из 2+ белых клеток подряд

**Пример 1 - 5×5:**

```text
_____
_____
_____
_____
_____
```

Вывод:

```text
TURFS
ALERT
STRIA
TRUST
YANKS
```

**Пример 2 - с чёрными клетками:**

```text
***___
______
______
______
***___
```

Вывод:

```text
***OBE
BRIARS
AORTAS
STEEVE
***NOS
```

## Алгоритм

Реализован через поиск с возвратом (backtracking) для задачи выполнения ограничений (CSP):

- **Эвристика MRV** (Minimum Remaining Values) - на каждом шаге заполняется слот с наименьшим числом подходящих слов-кандидатов, что сужает дерево перебора
- **Позиционный индекс** - для каждой длины слова и позиции буква отображается в множество подходящих слов. Поиск кандидатов сводится к пересечению множеств за O(1) на каждое ограничение
- **Упорядочивание по кроссируемости** - кандидаты сортируются по тому, насколько часто их буквы встречаются на соответствующих позициях в других словах - наиболее совместимые слова проверяются первыми
- **Случайные рестарты** - если поиск заходит в тупик, сетка сбрасывается и кандидаты перемешиваются заново. На каждую следующую попытку даётся в два раза больше времени

## Производительность

| Размер сетки                 | Типичное время |
| ---------------------------- | -------------- |
| 3×3 - 5×5                    | 500-1000 мс    |
| 6×N / N×6 с чёрными клетками | 8–17 с         |
| 7×7 с чёрными клетками       | 5–6 с          |

> **Нюансы**
> полные сетки 6×6 и 7×7 без чёрных клеток практически нерешаемы с обычным словарём, т.к. это даже математически сложная задача ([задача составления «слово-квадрата»](https://digitalcommons.butler.edu/wordways/vol8/iss4/2/)). Для больших сеток рекомендуется расставлять чёрные клетки, разбивая поле на более короткие слоты.

## Тестирование

```sh
node test.js
```

Запускает:

- **Unit-тесты** - проверка отдельных функций: `loadWords`, `buildIndex`, `parseGrid`, `extractSlots`, `getCandidates`, `isSlotFilled`, `isSlotValid`
- **Интеграционные тесты** - проверка целых сеток: валидация всех слов, отсутствие дубликатов
- **Замеры производительности** - время решения для разных размеров сетки
